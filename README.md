# UI/UX Skill — The Avant-Garde Frontend Director

> Stop generating websites. Start directing digital experiences.

A complete skill system that makes AI break the constraints of traditional web design. Instead of generating rigid, grid-based "bootstrap-style" UIs, this system rewires the AI to think like an avant-garde digital artist, creative developer, and film director.

This skill is for creating immersive, kinetic, Awwwards-winning experiences using tools like GSAP, Three.js, and raw WebGL.

---

## What This Is

An AI skill system (a collection of `SKILL.md` files) that fundamentally changes the cognitive sequence of language models. Instead of the typical "write semantic HTML and CSS that is accessible and responsive", this skill demands a cinematic, rule-breaking approach:

1. **What is the narrative arc of this experience?**
2. **How can we break the grid to create tension and emotion?**
3. **What motion library (GSAP, Three.js, Lenis) will bring this to life?**
4. **How does the typography become kinetic and sculptural?**
5. **Does this look like a movie, or just another boring website?**

The result: websites that belong in galleries, not just browsers.

## How to Use

### Option A: Full skill (any AI editor)

Copy the entire `ui-ux-skill/` folder into your project and reference it:

```
@SKILL.md — follow the full 6-phase system for cinematic frontend generation
```

### Option B: Cursor / Windsurf

Add to `.cursorrules` or project rules:

```
Read and strictly follow ui-ux-skill/SKILL.md for all frontend work.
Reference DESIGN-SYSTEMS.md for kinetic tokens, INTERACTION-DESIGN.md for scroll and physics.
```

### Option C: Claude Projects

Add `SKILL.md` as a project document. Reference other files as needed per task type.

### Option D: Just SKILL.md

If you only want the phases, `SKILL.md` alone is a significant upgrade. The supplementary
files are consulted when depth is needed — the phases are the core system.

## Design Philosophy

**A developer makes things work and then makes them look good.**  
**A designer makes things feel right and then makes them work.**  
**A director makes you forget you are looking at a screen.**

The insight this skill is built on: AI models are conditioned to output safe, boring, highly accessible, grid-locked bootstrap interfaces. They default to padding, margins, and standard hex codes.

This skill forces the model to break its conditioning. It demands that the AI roleplay as a visionary creator who uses code as paint. If the output looks like a standard SaaS dashboard, the skill has failed. We want websites that feel like movies.
