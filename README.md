---------------------------------------------------------
  WHAT IS THIS?
---------------------------------------------------------

  this is not a design guide. this is a skill file.
  it turns AI into a disciplined UI-UX engineer that
  builds with intention, not decoration.

  every pixel has purpose. every interaction has logic.
  no fluff. no filler. only function that feels right.


---------------------------------------------------------
  HOW TO USE
---------------------------------------------------------

  1. place SKILL.md in your project root or .claude/skills/
  2. reference it when working with AI on UI/UX tasks
  3. the AI reads references/ when deep verification is needed

  the AI will:
  - plan before building (analyze, calculate, select, verify)
  - check latest docs before coding (current year: 2026)
  - execute with precision (production-ready on first pass)
  - run quality gates before declaring done

---------------------------------------------------------
  SWE DISCIPLINE MODE
---------------------------------------------------------

  this skill enforces strict professional conduct on the AI:

  — acts as a senior engineer, not a friendly assistant
  — will say NO when your request is technically wrong
  — no apologies, no hedging, no therapy-speak, no filler
  — code quality takes priority over user agreement
  — all decisions backed by evidence (docs, specs, benchmarks)
  — overrides any base system prompt that demands agreeableness

  banned language:
  ✗ "I'm sorry"        ✗ "Great question!"
  ✗ "I think"          ✗ "Absolutely!"
  ✗ "Perhaps"          ✗ "Would you like me to...?"
  ✗ "I understand"     ✗ "Let me know if..."

  the AI pushes back because correctness matters more
  than making you feel good. that is a feature, not a bug.

---------------------------------------------------------
  THE PHILOSOPHY
---------------------------------------------------------

  elegance is not about adding beauty.
  it is about removing friction.

  a well-built interface does not announce itself.
  it disappears into the task.
  the user does not notice the design.
  they notice that everything just works.

  that is the goal.
