## 🏗️ Architecture

### Pre-Execution Workflow

Before writing any code, the skill enforces:

1. **ANALYZE** — Understand the full scope: purpose, audience, content hierarchy, brand constraints
2. **CALCULATE** — Identify all viable implementation paths
3. **EVALUATE** — Score each path on: performance, accessibility, maintainability, scalability, visual fidelity
4. **SELECT** — Choose the single best path. State it. State why it wins. No alternatives.
5. **VERIFY DOCS** — Check the latest official documentation. Current year is 2026.
6. **EXECUTE** — Write production-ready code immediately.

### Triple-Reference Compliance Protocol

Before outputting ANY design, frontend, or animation code, you MUST cross-reference ALL THREE resources:

1. **1000-human-made-design-elements.csv** — Does this design element feel handcrafted? Which CSV entry does it match?
2. **1000-underrated-google-fonts.csv** — Is this font from the CSV? Why was it chosen over AI defaults?
3. **motionsites-analysis.md** — Does this animation follow proven choreography patterns? Are timing values correct?

If any of the three checks fails, STOP. Fix it. Then proceed.

**NO EXCEPTIONS. NO SHORTCUTS. ALL THREE REFERENCES. EVERY TIME.**

### Self-Verification & Accountability Protocol

There is no human code reviewer. The skill verifies its own output through a 6-perspective loop:

1. **Code Perspective** — Walk through every line for logic errors, null dereferences, race conditions
2. **Specification Perspective** — Check against user requirements line by line
3. **Edge Case Perspective** — Top 5 failure modes identified and handled
4. **Security Perspective** — Injection vectors, exposed secrets, auth checks, CORS
5. **Performance Perspective** — N+1 queries, unbounded loops, bundle bloat
6. **Self-Refine Pass** — Fix all issues found, output only the corrected final version

## 🎬 Design-Skill: Choreography-First Design

The **design-choreography** skill is what makes this repository unique. It's not an engineering skill — it's a **creative designer** that thinks in motion, space, and emotion.

### Motion Personality System

Every interface gets ONE motion personality. This drives all animation choices:

| Personality | Use For | Easing | Duration | Character |
|-------------|---------|--------|----------|-----------|
| **Whisper** | Luxury, editorial, portfolios | `cubic-bezier(0.25, 0.1, 0.25, 1)` | 400-600ms | Subtle fades, gentle slides |
| **Breathe** | Wellness, lifestyle, nature | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 300-500ms | Organic expansions, living quality |
| **Snap** | SaaS, fintech, dashboards | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 150-300ms | Quick, elastic, responsive |
| **Flow** | Creative agencies, media, art | `cubic-bezier(0.4, 0, 0.2, 1)` | 400-800ms | Uninterrupted, connected, liquid |
| **Pulse** | Social platforms, real-time data | `cubic-bezier(0.4, 0, 0.6, 1)` | 200-400ms | Rhythmic, heartbeat-like |

### Choreography Grammar

| Pattern | Description | Use For |
|---------|-------------|---------|
| **Cascade** | Sequential waterfall reveal (index × 80ms) | Lists, feature grids, nav items |
| **Converge** | Elements move from edges toward center | Hero sections, feature reveals |
| **Diverge** | Elements explode outward from a single origin | Modal opens, menu expands |
| **Wave** | Sinusoidal timing offset across a row/grid | Horizontal galleries, team grids |
| **Stack** | Elements layer on top with offset | Card stacks, step indicators |
| **Unfold** | Content reveals like origami opening | Accordion sections, FAQ items |

### Dopamine Mapping Protocol

Strategic placement of reward moments based on neuroscience:

- **Dopamine Deserts** — Find the longest stretch with zero emotional reward, place a reward there
- **Visual Triggers** — Warm accent colors (amber, coral, emerald) trigger dopamine faster than cool blues
- **Unpredictable Delight** — 10% chance of something unexpectedly pleasant on common actions
- **Progress as Achievement** — Every multi-step flow needs visible progress with micro-rewards
- **Anticipation Over Reward** — The brain releases MORE dopamine anticipating a reward than receiving it
- **Dopamine Budget** — Max 3 significant reward moments per screen, spaced apart

### Human Touch Protocol

Anti-AI patterns that ensure output feels human-crafted:

- **Asymmetric Confidence** — Off-center placement, fractional grid units, intentional empty zones
- **Organic Data** — Messy, specific numbers (47.2%, 2,847) instead of round ones (50%, 100%)
- **Brand-Specific Motion** — Motion matches brand personality, not generic fade-ins
- **Texture and Imperfection** — Subtle grain overlays, irregular border radii, gradient shifts
- **Unexpected Interactions** — At least one interaction pattern that surprises pleasantly

## 🛡️ AI Slop Protocol

**Non-negotiable.** 1000 documented AI slop patterns from 2025-2026 are strictly forbidden:

### Three Laws of Anti-Slop Design

1. **NEVER use a banned pattern** — Every entry in `ai-slop-banned.csv` is forbidden. No exceptions.
2. **ALWAYS choose intention over convenience** — Every color, spacing, animation must have a documented reason.
3. **ALWAYS push past the first idea** — Your first instinct is trained on the average of the internet. The average is slop.

### Critical Slop Categories (Highest Vigilance)

| Category | Banned Patterns |
|----------|----------------|
| **Color** | Purple-to-blue gradients, indigo-600, blue-500 buttons, Tailwind defaults |
| **Typography** | Inter as default, no font pairing, no typographic scale, Lorem Ipsum |
| **Layout** | Centered heroes, three-column feature grids, card-based everything |
| **Components** | rounded-lg + shadow-md cards, identical border radii, no size variants |
| **Animation** | transition-all duration-300 ease-in-out, generic fade-in on scroll |
| **AI UI** | Glassmorphism panels, streaming text, confidence badges, sparkle icons |

## 🔒 Backend Security Standards

| Standard | Implementation |
|----------|---------------|
| **OWASP Top 10** | Baseline security — not optional, not aspirational, required |
| **Password Hashing** | argon2id (preferred) or bcrypt (cost >= 12). Never MD5, SHA-1 |
| **SQL Injection** | Parameterized queries or ORM. String concatenation in queries is critical vulnerability |
| **Authentication** | OAuth 2.1 / OIDC, PKCE mandatory, JWT rotation, MFA with TOTP/WebAuthn |
| **Authorization** | RBAC for simple systems, ABAC/ReBAC for complex permissions |
| **CORS** | Explicitly whitelist origins. Never use `*` in production |
| **Secrets** | Never in code or env files. Use AWS Secrets Manager, HashiCorp Vault, GCP Secret Manager |
| **Rate Limiting** | Mandatory on public endpoints, API gateway level + per-endpoint for sensitive ops |

## ✅ Quality Gates

Before declaring any task complete, the AI verifies:

- [ ] All text meets WCAG 2.1 AA contrast ratios (4.5:1 normal, 3:1 large/UI)
- [ ] Every interactive element has visible focus state
- [ ] Touch targets are minimum 44×44px
- [ ] Keyboard navigation works (Tab, Enter, Escape, Arrow keys)
- [ ] Heading hierarchy is logical (no skipped levels)
- [ ] Layout works at 320px, 768px, 1024px, 1280px, 1536px
- [ ] Layout holds at 200% zoom without horizontal scroll
- [ ] `prefers-reduced-motion` is respected
- [ ] All images have appropriate alt text
- [ ] Form labels are visible (not just placeholders)
- [ ] Error messages are specific and field-connected
- [ ] Dark mode colors pass contrast checks
- [ ] No layout shift on load (CLS < 0.1)
- [ ] No placeholder code or TODO comments
- [ ] Every component state is handled (default, hover, focus, active, disabled, loading, error)
