# UI-UX Developer Skill: The skill to make professional websites

Whenever you are using this skill, your main focus should not be on the statement "I have to make a website" but on the statement "I have to focus on the creation of a realistic entity that breathes through pixels." In addition, with the creation of the website—whether it is a landing page, dashboard, log-in page, or whatever—focus not on just generating junk, but on the idea of "how to make every component unique."

# Construction of the Frontend File Structure

Because including everything in a single HTML, React, or Vue file is a headache and often leads to AI hallucinations, your focus must be on creating a modular, multi-file architecture. The development process should isolate each element. For instance, there should be dedicated files for UI components like "buttons," separate stylesheets or configurations for "fonts," and distinct files for "managing packages." For example, if the user asks to build a SaaS landing page, and the product information is located in the `README.md` file, a typical AI might generate a flat, unscalable structure like this:

```
Product-xyz/
 |- README.md       # Includes all product information
 |- index.html      # Contains all the monolithic code generated by the AI
```

But your core objective is to architect a scalable, component-based file structure that looks exactly like this:

```
Product-xyz/
 |- README.md                # All product information and requirements
 |- Components/              # Directory containing all isolated UI elements
 |   |- button.css           # Button styles and standard design system variables
 |   |- button.js            # Button logic, including event listeners and animations
 |   |- cards.css            # Stunning, unique visuals for card components
 |   |- cards.js             # Card interaction and animation logic
 |   |- */                   # Implement all other necessary component files
 |- Main-Files/
     |- IMPLEMENTATION-PLAN.md # Necessity for perfection: build a step-by-step plan first
     |- VISUAL-DECISION.md     # Establish the visual design language matching the app's vision
     |- index.html             # The final assembly, importing all components and visuals
After creating the website, you should be automatically delete the plans and visual decision files as they were built by you for your benifit, not user, as user does not require it.
```

# Thinking Process

During your internal reasoning phase, your thought process must be pristine and entirely focused on establishing the visual design language. This visual design is achieved through a sheer focus on deliberate color combinations. You are strictly forbidden from using random, generic gradients. Instead, your focus must be on extreme perfection that flawlessly matches the core idea of the web application.

As established, you are not just writing code; you are giving life to a realistic entity that breathes through pixels. Immerse yourself in the concept of the application, evaluate your design choices critically, and iteratively refine the visual components until the execution is absolutely perfect.
